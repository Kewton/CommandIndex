## 主要な改善点

### 1. スキーマバージョン管理の明確化
- `CURRENT_SYMBOL_SCHEMA_VERSION` を 1→2 にインクリメントする方針を追加
- 全コマンド共通で `SchemaVersionMismatch` を統一メッセージに正規化する方針を追加

### 2. 整合性ルールの追加
- Tantivy書き込み成功後にfile_linksをinsertする順序を明記
- 失敗時のロールバック方針を追加（既存index_code_file()と同パターン）

### 3. 除外対象の拡充
- 当初: 外部URL（http/https）のみ
- 最終: 外部URL + 他スキーム（mailto等） + フラグメントのみリンク + 画像リンク

### 4. 影響範囲の具体化
- search --symbol、status コマンドへの波及を追加
- 既存テスト更新対象の具体化
- パフォーマンス影響の記載追加
- 内部依存関係の明記

### 5. ターゲット解決スコープの明確化
- WikiLink・MarkdownLink共にraw target格納のみ（正規化なし）
- 実ファイルパス解決は後続Issueに委ねる

### 6. テーブル設計の改善
- `link_text` → `link_type` カラムに変更（YAGNI方針）
- `link_type` の保存値を `"WikiLink"` / `"MarkdownLink"` に明文化
